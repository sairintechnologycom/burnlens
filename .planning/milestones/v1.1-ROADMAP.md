# Milestone v1.1: Billing & Quota

**Status:** ✅ SHIPPED 2026-04-30
**Phases:** 6–10
**Total Plans:** 31

## Overview

Make the Paddle-backed plans real — surface the user's tier, let them manage billing in-app, and enforce plan limits so Free/Cloud/Teams users can't exceed what they paid for. v1.1 is soft enforcement only: seat/API-key 402s + retention pruning + 80%/100% usage warnings. Hard ingest rejection deferred to v1.2.

## Milestone Summary

**Timeline:** 2026-04-18 → 2026-04-30 (12 days, 178 commits)
**Stats:** 241 files changed, +48,179 / -1,478 lines

**Key Decisions:**
- v1.1 is soft enforcement only — seat/API-key 402s + retention pruning + 80%/100% warnings; hard ingest 429 deferred to v1.2
- `plan_limits` Postgres table is the single source of truth; per-workspace overrides via JSONB merge
- Paddle webhooks are authoritative for plan state — app reads, never computes from checkout redirect
- Entitlement middleware on gated API routes is mandatory; UI gating alone is insufficient
- POLL_INTERVAL_MS flipped 30s→60s for UsageMeter to avoid double-poller scenario

**Issues Resolved:**
- Paddle paused event collapsed into canceled (D-23) — simpler state machine
- `_load_billing_summary` in Phase 8 mutation responses does not refresh Phase 10 subobjects — frontend re-polls within 60s (acceptable per D-22)
- PDL-02 partial: `transaction.completed` recovery relies on `subscription.updated` with status=active; documented design choice in 07-02 SUMMARY

**Issues Deferred:**
- Hard ingest 429 on over-quota (v1.2)
- Phase 9 VERIFICATION.md (covered by VALIDATION.md + 36 passing tests)
- test_plan_limits.py dotenv isolation shim (3-line fix, CI reliability gap)
- 25 human-UAT items (live Paddle sandbox + browser visual checks)

**Technical Debt Incurred:**
- BillingSummary TS interface omits scheduled_plan/scheduled_change_at (uses `as unknown as` cast in settings/page.tsx)
- `_PLAN_PRICE_CENTS` module-level constants are a v1.2 candidate for plan_limits column
- Phase 9 VERIFICATION.md absent — procedural debt only, substantive coverage closed by VALIDATION + tests

**Known Gaps (acknowledged at milestone close 2026-04-30):**
