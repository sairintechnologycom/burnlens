## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap inside .btn, chip internal padding |
| sm | 8px | Row gaps, sidebar-item padding, chip list gap |
| md | 16px | Cell padding in .data-table td/th, card internal padding |
| lg | 24px | .shell-content padding (already set globally), section gap |
| xl | 32px | .setup-card padding, modal internal padding |
| 2xl | 48px | .empty-state padding |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Toggle pill: 20px wide × 12px tall track; 8px × 8px thumb — matches
  existing .theme-toggle visual language, implemented with inline style
  (thumb fits cleanly in 12px tall track with 2px top/bottom margin).
- Touch targets for toggle button: minimum 44px height row (achieved via
  12px top + 12px bottom td padding = 24px + content height, acceptable for
  desktop-first dashboard).
- Edit modal: max-width 440px, matching existing .setup-card pattern.

Source: globals.css shell-content (24px), data-table (12px/16px), setup-card (32px), empty-state (48px).

```

  <div>                          ← direct child of shell-content (padding: 24px already applied)
    <!-- Stat strip (2 cells) -->
    <div class="stat-strip">
      <div class="stat-cell">    ← "Total Rules" count
      <div class="stat-cell">    ← "Enabled" count
    </div>

    <!-- Rules table card -->
    <div class="card" style="margin-top: 24px">
      <div class="section-header">
        <span class="section-header-title">Alert Rules</span>
        <!-- no section-header-action: viewer role has no add action -->
      </div>
      <table class="data-table">
        <thead>
          <tr>
            <th>Threshold</th>
            <th>Channel</th>
            <th>Slack</th>
            <th>Extra Emails</th>
            <th>Enabled</th>
            <th></th>            ← Actions column (Edit Rule button — owner only)
          </tr>
        </thead>

          <tr> ... </tr>         ← one row per alert_rule, ordered by threshold_pct ASC
```

Stat strip: 2 cells only (Total Rules, Enabled count). Grid is
`grid-template-columns: repeat(2, 1fr)` overriding the default 4-col strip
via inline style or a wrapper. Values use .stat-value class (28px tabular
numeric). Labels use .stat-label (12px --muted).

## Interaction Contract

### 1. Page load
- On mount: `setLoading(true)`, call `apiFetch("/api/v1/alert-rules", session.token)`.
- While loading: render 2 skeleton rows (`.skeleton` divs, height 40px, margin-bottom 8px)
  inside the table card area.
- On success: render stat strip + table. If empty array: render EmptyState.
- On error: render `.error-inline` with text "Couldn't load alert rules — retry" with
  onClick re-fetch.

### 2. Enable/disable toggle (owner only)
- Trigger: click toggle button in Enabled column.
- Optimistic update: immediately flip `rule.enabled` in local state before API call.
  Keep a copy of the pre-flip value: `const prev = rule.enabled`.
- API call: `PATCH /api/v1/alert-rules/{id}` with body `{ enabled: !prev }`.
- On success: show toast "Alert rule updated" (success).
- On error: revert state to `prev`, show toast "Failed to update rule" (error).
- While pending: toggle button disabled (pointer-events: none, opacity: 0.6).
- Role gate: if `session.role !== "owner"`, render toggle as read-only visual
  (no button, just a static colored dot — green enabled, dim disabled).

### 3. Edit rule (owner only)
- Trigger: click "Edit Rule" button (`.btn`, BellIcon 14px + "Edit Rule" label) in Actions column.
- Opens inline modal (not a new route). State: `const [editingRule, setEditingRule] = useState<AlertRule | null>(null)`.
- Modal pre-populates: threshold_pct select to current value, extra_emails chips to current array.
- Threshold select: `<select class="form-input">` with exactly two options: "80%" (value=80) and
  "100%" (value=100). No free-text input.
- Extra email chips:
  - Render each email in `extra_emails` array as a chip with × remove button.
  - Text input below chips: `onKeyDown` — when `e.key === "Enter"`, call
    `e.preventDefault()` then validate email (must contain "@"), add to local chip array.
  - Remove chip: filter chip from local array.
  - Duplicate prevention: skip add if email already in array (case-insensitive).
  - Validation: must contain "@" and "."; show inline error text (12px, --red) if invalid
    and Enter is pressed: "Enter a valid email address".
- Save: PATCH `/api/v1/alert-rules/{id}` with `{ threshold_pct, extra_emails }`.
  Both fields always sent (full state, not diff) — backend handles partial update.
- While saving: "Save Changes" button disabled, text changes to "Saving…".
- On success: close modal, update local rule state, show toast "Alert rule saved" (success).
- On error: keep modal open, show toast "Failed to save rule" (error). Do not revert
  threshold_pct / extra_emails local state (user can retry without re-entering data).
- Cancel: close modal, discard local modal edits (do not mutate parent state).
- Close on overlay click: yes. Close on Escape keydown: yes
  (`useEffect(() => { const handler = (e) => { if (e.key === "Escape") setEditingRule(null); }; ... }`).

### 4. Role-gated rendering
- Viewer role (`session.role === "viewer"`):
  - Enabled column: static colored dot only (no button).
  - Actions column: not rendered (no th, no td).
  - Edit modal: not accessible.
- Owner role:
  - Full toggle buttons + Actions column + Edit Rule button.

### 5. Slack webhook link
- Modal footer link: "Manage Slack webhook in Settings →"
- Rendered as `<a href="/settings">` styled font-mono 12px, color var(--cyan),
  text-decoration underline. Closes modal on click (navigate away).

### 6. Sidebar nav
- Add `{ href: "/alerts", label: "Alerts" }` to "Intelligence" group in Sidebar.tsx,
  after the "Budgets" entry.
- Icon: `<BellIcon size={15} />` from lucide-react.
- No `lockedForPlan` — alerts visible to all paid plans.

| State | What to render |
|-------|----------------|
| Loading | 2× `.skeleton` rows (height: 40px) inside card, 2× `.skeleton` stat cells (height: 48px) |

| Modal save error | Toast error, modal stays open |

## Accessibility Notes

- Toggle buttons must have `aria-label` set dynamically ("Enable alert rule" when disabled,
  "Disable alert rule" when enabled) and `aria-pressed={rule.enabled}`.
- Modal must trap focus: first focusable element on open is the threshold select.
  Escape closes modal (keyboard contract above).
- Email chip remove buttons: `aria-label="Remove {email}"`.
- All interactive table cells: `<td>` contains a `<button>` — not raw click handlers on `<td>`.
- Color is not the only signal: enabled/disabled toggle has aria-pressed; threshold badge
  has text value alongside color.
