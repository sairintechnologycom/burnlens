## Security Threat Model (ASVS L1)

| Threat | Severity | Mitigation |
|--------|----------|-----------|
| Token hash collision allows account takeover | HIGH | `token_hash TEXT NOT NULL UNIQUE` — duplicate inserts fail at DB level; tokens use `secrets.token_urlsafe(32)` → 256 bits entropy (Plan 03) |
| Expired token reuse | HIGH | `expires_at TIMESTAMPTZ NOT NULL` — all queries include `AND expires_at > now()` (enforced in Plan 03) |
| Deleted-user tokens orphaned | LOW | `ON DELETE CASCADE` — tokens auto-removed when user is deleted |
| Schema injection via migration | LOW | All DDL uses parameterized `CREATE TABLE IF NOT EXISTS` literals — no user input in schema statements |

No high-severity unmitigated threats. Schema changes only; no request-handling logic in this plan.
</threat_model>
