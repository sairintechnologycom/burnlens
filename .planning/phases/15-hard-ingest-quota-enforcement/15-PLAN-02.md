    ```python
    async def _check_quota_or_raise(
        workspace_id: str,
        plan: str,
        batch_size: int,
    ) -> None:
        """Pre-insert quota gate — raises HTTPException(429) if any quota dimension is breached.

        Must be called AFTER auth (workspace_id known) and BEFORE execute_bulk_insert.
        Do NOT wrap in try/except — the 429 must propagate to FastAPI's response handler.

        Checks in order: requests → tokens → spend_usd → seats.
        None cap on any dimension means unlimited — that dimension is skipped.

        Race condition: read-then-check has a small overshoot window. This is accepted
        per Phase 9 precedent (see ingest.py comment at _record_usage_and_maybe_notify).
        The alternative (SELECT FOR UPDATE) would serialize all ingest through one DB row lock.

        QUOTA-04 interpretation: "seat" enforcement is active-member-count vs seat_count limit.
        api_keys has no seat_number column; member count is the natural proxy (A2 from RESEARCH.md).
        """
        limits = await resolve_limits(workspace_id)
        if limits is None:
            return  # workspace passed auth; resolve_limits None is a belt-and-suspenders guard

        # --- QUOTA-01: monthly request cap ---
        if limits.monthly_request_cap is not None and limits.monthly_request_cap > 0:
            usage_rows = await execute_query(
                """
                SELECT COALESCE(SUM(request_count), 0) AS current_count
                FROM workspace_usage_cycles
                WHERE workspace_id = $1 AND cycle_end > NOW()
                """,
                workspace_id,
            )
            current_requests = int(usage_rows[0]["current_count"]) if usage_rows else 0
            if current_requests >= limits.monthly_request_cap:
                raise HTTPException(
                    status_code=429,
                    detail=QuotaExceededDetail(
                        dimension="requests",
                        current=float(current_requests),
                        limit=float(limits.monthly_request_cap),
                    ).model_dump(),
                )

        # --- QUOTA-02: monthly token cap ---
        if limits.monthly_token_cap is not None and limits.monthly_token_cap > 0:
            token_rows = await execute_query(
                """
                SELECT COALESCE(SUM(token_count), 0) AS current_count
                FROM workspace_usage_cycles
                WHERE workspace_id = $1 AND cycle_end > NOW()
                """,
                workspace_id,
            )
            current_tokens = int(token_rows[0]["current_count"]) if token_rows else 0
            if current_tokens >= limits.monthly_token_cap:
                raise HTTPException(
                    status_code=429,
                    detail=QuotaExceededDetail(
                        dimension="tokens",
                        current=float(current_tokens),
                        limit=float(limits.monthly_token_cap),
                    ).model_dump(),
                )

        # --- QUOTA-03: cumulative spend ceiling ---
        if limits.monthly_spend_cap_usd is not None and limits.monthly_spend_cap_usd > 0:
            spend_rows = await execute_query(
                """
                SELECT COALESCE(SUM(spend_usd), 0) AS current_count
                FROM workspace_usage_cycles
                WHERE workspace_id = $1 AND cycle_end > NOW()
                """,
                workspace_id,
            )
            current_spend = float(spend_rows[0]["current_count"]) if spend_rows else 0.0
            if current_spend >= float(limits.monthly_spend_cap_usd):
                raise HTTPException(
                    status_code=429,
                    detail=QuotaExceededDetail(
                        dimension="spend_usd",
                        current=current_spend,
                        limit=float(limits.monthly_spend_cap_usd),
                    ).model_dump(),
                )

        # --- QUOTA-04: seat cap (active member count vs plan seat_count) ---
        if limits.seat_count is not None and limits.seat_count > 0:
            member_rows = await execute_query(
                """
                SELECT COUNT(*) as count FROM workspace_members
                WHERE workspace_id = $1 AND active = true
                """,
                str(workspace_id),
            )
            member_count = int(member_rows[0]["count"]) if member_rows else 0
            if member_count > limits.seat_count:
                raise HTTPException(
                    status_code=429,
                    detail=QuotaExceededDetail(
                        dimension="seats",
                        current=float(member_count),
                        limit=float(limits.seat_count),
                    ).model_dump(),
                )


    ```

## Trust Boundaries

| POST /v1/ingest → _check_quota_or_raise | Quota check is a resource access control boundary; workspace_id comes from authenticated lookup |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | Elevation of Privilege | _check_quota_or_raise — concurrent batch race | accept | Same read-then-check race accepted in Phase 9 (ingest.py comment). Lock-free UPSERT; small overshoot is bounded and acceptable. SELECT FOR UPDATE would serialize all ingest through a single row lock — rejected per Phase 9 decision. |
| T-15-02 | Elevation of Privilege | QUOTA-04 seat cap — API key sharing | accept | Quota is per-workspace not per-key; sharing a key doesn't bypass workspace-level enforcement. Active member count is the authoritative seat check. |
| T-15-03 | Information Disclosure | QuotaExceededDetail 429 body | accept | Only current usage value and plan limit exposed. No workspace PII, no pricing tables, no internal identifiers. Structure mirrors existing 402 body in api_keys_api.py. |
| T-15-04 | Denial of Service | _check_quota_or_raise adds 2–4 extra DB reads per ingest | accept | Each read is a simple indexed SELECT on workspace_usage_cycles (indexed by workspace_id) or workspace_members. Expected p99 < 5ms per read at Railway Postgres scale. Alternative: single combined query — deferred as premature optimization. |
| T-15-05 | Tampering | batch_tokens / batch_spend_usd aggregation in ingest() | mitigate | Values derived from Pydantic-validated request.records fields (input_tokens: int, cost_usd: Decimal). Pydantic rejects non-numeric values at deserialization. No freeform string injection path. |
