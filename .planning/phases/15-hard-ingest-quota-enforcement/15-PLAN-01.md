|----------|-------------|
| limit_overrides JSONB → resolve_limits() | Workspace-level override values come from authenticated admin writes; SQL function casts them safely with explicit ::type casts |
| plan_limits defaults → workspace | Plan seeds are NULL (unlimited); no default enforcement until caps are set explicitly |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | Elevation of Privilege | resolve_limits() SQL function | mitigate | COALESCE pattern ensures override only applies when value is a valid castable string; invalid override value causes NULL fallback to plan default, not bypass |
| T-15-02 | Tampering | plan_limits column defaults (NULL) | accept | NULL = unlimited is intentional; caps must be explicitly set via admin or limit_overrides; no accidental enforcement on existing workspaces |
| T-15-03 | Information Disclosure | QuotaExceededDetail response body | accept | Only current usage count and plan limit exposed — no workspace PII, no internal schema details, no pricing data |
