# Release & Safety Checklist — BurnLens

Use this checklist to verify the system before releasing or deploying new changes.

## 1. Local Code Quality Checks
- [ ] Code is fully typed and `mypy` type-checks clean.
- [ ] No syntax errors or warnings generated by `ruff` or `eslint`.
- [ ] PII safety check: ensure no raw prompts, raw keys, or secret tokens are logged.

## 2. Test Suite Validation
- [ ] Downgrade click to stable target:
  ```bash
  uv pip install click==8.1.8
  ```
- [ ] Run complete Python test suite:
  ```bash
  .venv/bin/pytest
  ```
- [ ] Check API contract between frontend and backend:
  ```bash
  BURNLENS_PYTHON=../.venv/bin/python3 npm run contract:snapshot
  npm run test
  ```

## 3. Proxy Hot-Path Smoke Checks
- [ ] Run proxy: `python -m burnlens start`
- [ ] Forward non-streaming request:
  - Verify upstream response returns 200.
  - Verify request records in SQLite with correct input/output tokens and cost.
- [ ] Forward streaming request:
  - Verify chunks stream back with no buffering latency.
  - Verify final usage is captured and recorded.
- [ ] Verify hard budget block:
  - Trigger daily budget cap limit.
  - Verify proxy returns HTTP 429 immediately *before* sending request upstream.

## 4. Deployment Gateways (SaaS Backend)
- [ ] Local migrations check: run schema checks on local PostgreSQL instance.
- [ ] Verify CORS preflight handling:
  - Run staging tests to confirm rotated domains or serverless restarts don't drop CORS headers.
- [ ] Build Next.js site locally:
  ```bash
  npm run build
  ```
- [ ] Deploy staging build to Vercel/Railway and verify signup/login end-to-end.
